# Code of Conduct

## Our Pledge

Quantiles is an open-source project for AI evaluation infrastructure. We want the project to be useful, technical, and welcoming for contributors with different backgrounds, experience levels, and perspectives.

We expect everyone participating in the Quantiles community to act with professionalism, respect, and care.

This Code of Conduct applies to all project spaces, including GitHub issues, pull requests, discussions, documentation, examples, agent-facing files, community channels, events, and any other spaces where people represent the project.

## Expected Behavior

Examples of behavior that contributes to a healthy project community include:

- Using clear, respectful, and constructive communication
- Giving and receiving technical feedback in good faith
- Focusing criticism on ideas, code, documentation, benchmarks, and design decisions rather than people
- Being patient with contributors who are learning the project
- Asking clarifying questions before assuming intent
- Respecting maintainers’ time and review process
- Keeping discussions relevant to the issue, pull request, or project area
- Documenting decisions when they affect users, contributors, or coding agents
- Acknowledging mistakes and correcting them when possible

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or sustained disruption
- Personal attacks, insults, or demeaning comments
- Discriminatory language or behavior
- Sexualized language, imagery, or attention
- Trolling, deliberate provocation, or repeated off-topic arguments
- Publishing or threatening to publish someone’s private information
- Dismissing or mocking contributors based on experience level, background, identity, or communication style
- Misrepresenting project status, benchmark results, security issues, or contributor work
- Using project spaces to promote unrelated commercial, political, or personal agendas
- Any other conduct that maintainers reasonably determine to be inappropriate for a professional open-source project

## Technical Discussion Standards

Quantiles is an engineering project. Technical disagreement is expected and useful when handled well.

When discussing technical decisions:

- Prefer evidence over assertion
- Use reproducible examples when possible
- Separate correctness, design preference, and implementation detail
- Be explicit about tradeoffs
- Avoid overstating benchmark results or evaluation claims
- Document compatibility, reproducibility, and provenance concerns
- Keep pull request feedback actionable
- Avoid blocking discussions with repeated objections after a maintainer decision has been made

Good technical feedback is specific, respectful, and tied to the project’s goals.

## AI and Coding Agent Use

Quantiles is designed to work well with coding agents. Contributors may use coding agents to draft code, tests, documentation, benchmarks, or issue responses.

When using coding agents, contributors remain responsible for the final contribution.

Agent-assisted contributions should:

- Be reviewed by the contributor before submission
- Follow `AGENTS.md`, `CONTRIBUTING.md`, and project documentation
- Include tests when behavior changes
- Avoid fabricated citations, benchmark claims, APIs, or implementation details
- Avoid committing secrets, private data, or generated artifacts that do not belong in the repository
- Clearly update documentation when agent-facing behavior changes

Do not use coding agents to spam issues, generate low-quality pull requests, impersonate maintainers, or bypass project review expectations.

## Maintainer Responsibilities

Maintainers are responsible for clarifying standards of acceptable behavior and may take appropriate action in response to behavior that violates this Code of Conduct.

Maintainers may:

- Edit or remove inappropriate comments, commits, issues, pull requests, discussions, or documentation
- Ask contributors to revise language, tone, or behavior
- Close issues or pull requests that are hostile, disruptive, or not constructive
- Temporarily or permanently restrict participation in project spaces
- Escalate serious violations to the relevant platform or organization

Maintainers should apply this Code of Conduct consistently and fairly.

## Reporting

If you experience or observe unacceptable behavior, please report it to the maintainers.

Do not report Code of Conduct incidents through public GitHub issues.

Send reports to:

```text
conduct@quantiles.io
```

A report should include as much relevant information as possible:

- What happened
- Where it happened
- When it happened
- Who was involved
- Links, screenshots, or references if available
- Whether the behavior is ongoing
- Any safety or privacy concerns

Maintainers will handle reports with care and will limit information sharing to what is necessary to review and respond to the report.

## Enforcement

Maintainers may take one or more of the following actions depending on the situation:

1. **Correction**  
   A private or public request to change language, behavior, or approach.

2. **Warning**  
   A formal warning that continued behavior may result in further action.

3. **Temporary restriction**  
   A temporary restriction from issues, pull requests, discussions, or other project spaces.

4. **Permanent restriction**  
   A permanent restriction from participating in the project community.

5. **Escalation**  
   Reporting serious violations to GitHub, another platform, an employer, event staff, or legal authorities when appropriate.

The response will depend on the severity, impact, pattern, and risk of the behavior.

## Scope

This Code of Conduct applies within project spaces and when an individual is representing the Quantiles project in public or private spaces.

Examples of representing the project include:

- Using an official project account
- Acting as a maintainer or reviewer
- Speaking on behalf of the project
- Presenting project work at an event
- Participating in discussions where project affiliation is relevant

## Attribution

This Code of Conduct is inspired by widely used open-source community standards and adapted for Quantiles as an engineering-focused AI evaluation infrastructure project.
